shellr platform

The main project is a single-VM platform with deliberate service boundaries: public routing through Nginx, app runtimes in Docker containers, GitHub Actions deployment, observability, and backup. It is intentionally sized for one host and intentionally documented like something that may need to be handed over later.

Last reviewed: April 2026 Scope: live services + docs Review angle: problem, boundary, outcome
How to read these projects Each project is treated as a runtime decision, not just a code exercise. The relevant questions are what problem it solves, how it is isolated, and how it behaves operationally once deployed.
